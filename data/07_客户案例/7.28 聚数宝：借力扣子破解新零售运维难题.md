## 业务挑战

随着业务的快速扩张，聚数宝在保障服务承诺的过程中，⾯临着⽇益严峻的运维挑战，具体体现在以下四个⽅⾯：

- **运维时效压⼒剧增**

- 随着客户群体的持续扩⼤，聚数宝所覆盖的数据模块呈指数级增⻓。作为承诺数据完整、 时效、 准确的⾃动化SaaS系统，运维团队必须对数据获取流程中的任何故障做出快速响应和修复，这给前后端协作团队带来了巨⼤的履约压⼒。

- **故障类型多样，定位耗时冗⻓**

- 不同电商平台的业务规则和技术架构千差万别，导致⽇常数据抓取流程中产⽣ 的故障⼯单类型繁多、 成因复杂。运维⼈员需要针对海量平台持续积累故障处 理经验， 问题定位和排查的耗时成为影响效率的关键瓶颈。

- **平台频繁改版，运维压⼒陡增**

- 在电商⼤促等关键业务节点，各⼤电商平台通常会进⾏页⾯或接⼝的升级改版。这不仅导致故障⼯单批量涌现， 同时也恰逢客户对数据时效性要求最⾼的 时期，运维团队⾯临着需求与故障双重叠加的巨⼤压⼒。

- **协同流程分散，缺乏全局评估视角**

- 履约团队按客户维度划分，发现故障后独⽴提交⼯单。为追求单点时效，⼯单 处理流程被过度简化，导致多个客户遇到的同类根源问题被分散成独⽴⼯单，全部涌向运维⼯程师。这种模式缺乏对问题的全局分类与过滤机制，造成了⼤量重复劳动，也难以从宏观视角评估和定位共性问题。

## 解决方案

### Agent 开发

为应对上述挑战，聚数宝基于扣子编程构建了⼀套智能化的⾃动⼯单 Agent 解决⽅案，通过四个核⼼环节重塑了运维流程。

运维界面

1. **智能故障识别与初步定位**

- 故障识别 Agent 依托扣子编程构建的强⼤知识库，在接收到故障告警后，可⾃动读取流程故障⽇志。通过调⽤知识库开展深度语义理解，Agent 能够精准识别故障内容，并完成初步的问题定位，为后续处理提供决策依据。

2. **⾃动化⼯单分诊与⽅案提供**

- 自动工单 Agent 会智能判定故障的性质，明确其属于偶发性问题还是需要人工介入的重大故障。

- 针对偶发故障，Agent 会自动执行流程重试操作，并持续监测后续运行状态。

- 若判定需要工程师介入，它会自动创建并提交工单，同时附上详细的故障解读与定位分析结果，极大地助力工程师缩短问题解决时长。

3. **重复故障智能归并，减少冗余⼯作**

- 通过分析每日的工单历史数据，工单去重 Agent 能够自动识别并归类相似的故障工单。这一机制有效避免了由同一根源问题产生的重复工单流转至运维工程师处，既保障了故障得到高效处理，也显著减少了工程师的重复性工作，从而大幅提升了整体工单处理效率。

4. **全局故障洞察与主动提醒**

- 工单分析 Agent 会对所有故障信息进行全面、系统的分析，并定期输出流程运行稳定性报告。该报告能够揭示普遍存在的共性问题和潜在风险，为产品和研发团队优化产品性能、提升系统稳定性提供宝贵的数据洞察和决策支持。

### Agent 调优

Agent 开发是一个需覆盖全生命周期的系统工程，不同阶段需聚焦差异化重点以保障最终效果：

- 开发阶段需要提高提示词的编写效率、版本化管理 Prompt。

- 上线前要确保 AI 应用达到生产级要求。

- 上线后要进行线上监控、发现 Badcase、定位问题场景、识别 token 消耗和成本计费，并确定迭代方向。

为了 Agent 的输出效果可以实现持续的优化，聚数宝引入扣子罗盘作为 AgentOps 的平台。 在扣子罗盘的能力加持下，聚数宝以 “开发 + 调优” 并行投入的模式， 形成了从开发到评测、线上观测、效果优化再到开发迭代的新主流开发模式。

在整个 Agent 开发、调试的过程中， 扣子罗盘提供了 Prompt 智能优化、多模态评测、Trace 数据回流、消息评价（点赞点踩）等高级功能，整个调优的过程得到了有效提升，大幅度加速了 Aegnt 的上线速度及后续的持续优化的效率。
