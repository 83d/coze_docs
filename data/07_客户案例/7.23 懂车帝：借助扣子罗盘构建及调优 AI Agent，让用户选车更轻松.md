# 懂车帝：借助扣子罗盘构建及调优 AI Agent，让用户选车更轻松

懂车帝是字节跳动旗下的一站式汽车信息、交易与服务平台。为进一步优化用户体验，懂车帝基于 AI Agent 提供智能车型推荐、车型 PK、AI 选车等能力。基于扣子罗盘提供的实时全链路观测与节点性能监控等能力，懂车帝 AI Agent 构建及调优效率大幅提升。

## 客户介绍

懂车帝是字节跳动旗下的一站式汽车信息、交易与服务平台，凭借海量专业汽车资讯、全面车型数据与创新购车服务，在用户群体中树立起极高口碑。为进一步优化用户体验，懂车帝基于 AI Agent 提供智能车型推荐、车型 PK、AI 选车等能力，致力于打造更智能、便捷、个性化的汽车资讯及服务生态，全方位满足用户选车、购车、用车等环节的需求。

## 业务挑战

在构建与调优 AI Agent 的复杂进程中，懂车帝研发团队面临诸多关键难题。一方面，需要精准洞悉 Agent 各个关键节点的输入与输出情况，以此判断模型实际输出效果是否符合设计预期以及业务场景的真实需求，进而为 AI Agent 的调优工作提供真实、全面的线上运行数据支撑。另一方面，Agent 的性能优化与成本把控也是至关重要的环节。特别是在大模型节点，时常出现运行耗时过长甚至报错的状况，此时急需能够快速定位瓶颈点，并及时采取有效优化措施。

然而，懂车帝团队在调优 AI Agent 时，遭遇了传统观测系统的局限。传统观测系统并不完全适配大模型复杂场景，对于 Prompt 内容展示、大模型输入/输出、工具调用等核心关键环节，存在追踪与分析的重重困难。因此，团队只能借助离线数仓的方式，留存 Agent 线上运行数据，并编写脚本对模型输出数据与业务场景的匹配度进行回溯分析，以此指导 AI Agent 的下一步调优工作。但在此过程中，一系列问题逐渐凸显：

1. **关键环节回溯依赖人工且数据还原困难**：采用离线数仓方式，用户在处理数据时，面对的是杂乱无序的数据状态，难以精准还原实际业务现场，例如 Prompt Template 的完整结构无法精确重现。这使得从海量数据中梳理出有效线索，定位问题根源的工作变得异常艰巨。

2. **T + 1 数据排查严重制约调优与诊断效率**：受限于离线数仓的特性，研发人员仅能在 T + 1 天对前一日的线上数据进行回溯分析。这不仅导致线上即时出现的问题无法得到实时诊断，在实际开展 AI Agent 调优工作时，也造成了显著的效率损耗。

3. **性能与成本监控手段缺失**：传统观测手段难以精准展现 Agent 各节点的耗时情况，以及大模型运行过程中的 Token 消耗等关键数据指标，使得研发团队在性能优化与成本管控方面缺乏直观、有效的数据依据。

## 解决方案

扣子罗盘为懂车帝上述研发困境提供了行之有效的解决方案：

1. **实时全链路数据洞察**：扣子罗盘强大的 Trace 功能，能够提供完全适配 AI Agent 场景的全量数据，涵盖大模型节点的详细输入输出数据、各模型组件的调用情况等关键信息。借助这一功能，研发人员得以全面洞察 AI Agent 线上运行的所有数据以及关键节点状态，为 AI Agent 调优工作准备了充足、可靠的原始数据，从而能够有的放矢地深入研究，制定极具针对性的优化策略。

2. **节点性能精准监控**：扣子罗盘的 Trace 功能支持对 AI Agent 各节点的耗时情况、大模型 Token 消耗等关键数据进行精准查看。基于这些精确的数据结果，研发人员能够快速锁定性能瓶颈，高效解决问题，有力保障业务的稳定、高效运行。
