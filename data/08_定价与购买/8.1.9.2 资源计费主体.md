# 资源计费主体

资源计费主体指承担扣子相关费用的主体。 在扣子中，使用扣子资源产生的费用由资源的使用者承担，使用方舟模型产生的费用由方舟模型接入点的创建者承担。

| **费用类型** | **资源计费主体** |
| --- | --- |
| 扣子费用 | 由资源的使用者承担，资源的使用者是指实际调用或使用资源的扣子账号。  如果资源的使用者是子账号，则产生的账单由其对应的主账号支付。 |
| 方舟模型费用 | 方舟模型 Token 费用由方舟模型接入点的创建者承担。   如何区分方舟模型和扣子模型，可参考[如何在扣子中区分扣子模型和火山方舟模型](https://docs.coze.cn/coze_pro/model_fee)。 |

## 方舟模型费用

在扣子商店、模板库、调试区、飞书、微信、API 等所有渠道中产生的方舟模型 Token 费用，均由方舟模型接入点的创建者承担。

企业版（标准版、旗舰版）用户开通方舟模型资源、创建模型接入点之后，其拥有的所有工作空间中的任意用户都可以使用这些方舟模型，由模型接入点的创建者付费。关于方舟模型和扣子模型的定义及区别，可参考[如何在扣子中区分扣子模型和火山方舟模型](https://docs.coze.cn/coze_pro/model_fee)。

## 扣子编程费用

除方舟模型以外产生的扣子费用，均由资源的使用者承担，例如在扣子商店中和任意智能体对话，智能体的使用者承担相应的费用，通常包括扣子模型 Token 费、插件费用、音视频费用等。

扣子通过访问资源的扣子用户身份信息来判断资源的使用者。在各种不同的渠道中，使用者的身份详细说明如下：

> [!NOTE]
>
> **说明**
>
> - 如果智能体或应用中使用了方舟模型，那么在任意渠道使用该智能体或应用时产生的方舟模型 Token 费用，均由方舟模型接入点的创建者承担。
>
> - 企业版（标准版、旗舰版）、原团队版、原企业版成员积分消耗说明如下：
>
> - 内部成员（子用户）：在企业版生效期间，内部成员或尚未加入企业的子用户，在个人免费版中使用扣子时，均消耗企业版的积分，不会消耗个人版的积分。
>
> - 外部成员（访客）：在企业版生效期间，访客积分抵扣说明：
>
> - 访客从企业版切换到个人版后，消耗个人版积分。
>
> - 访客在企业中使用工作空间内的资源时，消耗企业版的积分。
>
> - 访客在企业商店、扣子商店和模板中使用时，消耗个人版积分。

| **渠道** | **使用者明细** |
| --- | --- |
| - 扣子商店   - 扣子模板  | 资源使用者为使用资源的用户**登录的扣子账号**。   例如在扣子商店和任意智能体对话，由当前登录的扣子账号承担扣子模型 Token 费用。  |
| 智能体或工作流调试  | 在智能体或工作流调试页面，包含如下两种情况：  - 如果使用的资源属于企业工作空间，那么资源使用者为资源所在**工作空间所有者的扣子账号**。例如访客在企业工作空间下的智能体调试页面与智能体对话，那么由该智能体所在工作空间的所有者的扣子账号承担扣子模型 Token 费用。  - 如果使用的资源属于个人版套餐中的工作空间，那么资源使用者为使用资源的用户**登录的扣子账号**。 例如在个人版工作空间下的智能体调试页面与智能体对话，那么由当前登录的扣子账号承担扣子模型 Token 费用。  |
| - API 或 SDK   - 抖音小程序   | 资源使用者为**访问密钥创建者**。   API 访问密钥的创建者指创建个人访问密钥或 OAuth 应用 App 的扣子用户。无论调用哪些资源，扣子将 API 访问密钥的创建者视为资源的使用者。  |
| - 微信小程序  - 微信服务号  - 微信客服  - 微信订阅号  - 飞书 | 资源的使用者为**智能体或应用所有者**。   在微信、飞书等渠道中，和智能体对话、使用扣子应用时，默认使用智能体或应用所有者的扣子身份信息发起请求，所以扣子将其视为资源的使用者。 |
| 飞书多维表格、掘金、豆包 | 资源使用者为**各个平台**。  在这些平台上使用任意智能体时，产生的费用均由对应平台（飞书多维表格、掘金或豆包）承担。 |
| 团队自定义渠道  | 资源使用者为**团队自定义渠道的创建者**。  在这些渠道中使用任意智能体时，默认使用渠道创建者的扣子身份信息，所以产生的费用均由渠道创建者承担。关于自定义渠道的定义，请参考[了解项目发布](https://docs.coze.cn/guides/publish_overview)。 |
| 公共渠道 | 不同公共渠道的计费主体不同，在添加公共渠道时，你可以在对应的公共渠道上获取计费主体。   <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/4be025d594ce439fac6337dcc88fad1d~tplv-goo7wpa0wc-quality:q75.awebp" width="373">   关于公共渠道的定义，请参考[了解项目发布](https://docs.coze.cn/guides/publish_overview)。 |

## 示例

### 发布到扣子商店

- 场景

- 开发者 A：购买个人进阶版套餐，开发并发布了一个智能体到扣子商店。

- 使用者 B：属于某企业版套餐的成员，在扣子商店与开发者 A 开发的智能体对话。

- 费用抵扣

- 开发者 A 在扣子商店与自己开发的智能体对话时，扣除开发者 A 的积分。

- 使用者 B 在扣子商店与开发者 A 开发的智能体对话时，扣除使用者 B 所在企业的积分。

### 发布到飞书多维表格

- 场景

- 开发者 A：属于某企业版套餐的成员，开发并发布了一个智能体到飞书多维表格。

- 使用者 B：在飞书多维表格中与开发者 A 开发的智能体对话。

- 费用抵扣

- 无论是开发者 A 还是使用者 B，在飞书多维表格中与上述智能体对话时，均扣除飞书多维表格平台的积分，不会扣除开发者 A 和使用者 B 的积分。

### 发布到微信

- 场景

- 开发者 A：购买个人进阶版套餐，开发并发布了一个智能体到微信。

- 使用者 B：在微信中与开发者 A 开发的智能体对话。

- 费用抵扣

- 不管是开发者 A 还是使用者 B，在微信中与上述智能体对话时，均扣除开发者 A 的积分。

## 常见问题

### 发布到商店的智能体由谁付费？

在扣子商店和任意智能体对话，由智能体使用者承担费用，通常包括智能体调用费和模型 Token 费。如果智能体中使用了方舟模型，则方舟模型费用由模型接入点的创建者承担。

例如开发者 A 创建了一个智能体，其中使用了扣子提供的模型，没有使用方舟模型。这个智能体发布到了商店供其他用户使用，扣子用户 B 在扣子商店中与这个智能体对话，会扣减用户 B 的积分，不会扣减开发者 A 的积分。

### 知识库空间由谁付费？

购买付费套餐后，当知识库空间的使用量超过免费额度时，则由知识库所在的订阅套餐的购买者支付费用。

### 在飞书使用智能体或应用，提示开发者余额不足，该如何充值？

在飞书使用扣子智能体或应用时，产生的费用由智能体/应用的开发者承担，如果提示余额不足，需要由智能体/应用的开发者购买积分，具体请参见[购买积分](https://docs.coze.cn/coze_pro/credits)。

### 为什么使用智能体时提示积分不足，但其他用户可以使用？

发布到扣子商店的智能体，资源计费主体为使用者。如果使用者账号有积分，那么他们仍可以使用对应的智能体。具体说明，请参考[资源计费主体](https://docs.coze.cn/coze_pro/payer)。
