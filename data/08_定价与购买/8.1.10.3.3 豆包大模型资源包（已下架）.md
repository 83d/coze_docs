# 豆包大模型资源包（已下架）

> [!NOTE]
>
> **说明**
>
> 2025 年 2 月 13 日扣子专业版调整计费规则，原有的豆包大模型资源包已下架，但已购买的资源包仍可继续使用，抵扣范围保持不变。如果你有新的服务需求，请及时购买扣子积分，详情请参考[积分](https://docs.coze.cn/coze_pro/credits)。

如果智能体使用豆包大模型，且未来一段时间内的 Token 用量可预估，那么可以按需购买豆包大模型资源包。相较于按量计费模式，豆包大模型资源包的性价比更高。

## 抵扣范围

扣子专业版的智能体默认使用火山引擎方舟模型服务提供的豆包模型。扣子专业版现推出豆包大模型资源包，可抵扣使用大语言模型推理服务时，豆包模型产生的 Token，包括输入 Token 和输出 Token。

> [!NOTE]
>
> **说明**
>
> - 豆包大模型资源包支持抵扣的模型范围为火山方舟提供的部分豆包模型，详细清单可参考[抵扣系数](https://docs.coze.cn/coze_pro/doubao_resource_pack)。
>
> - 豆包大模型资源包不仅限于抵扣扣子专业版的豆包模型 Token 用量，也会抵扣火山方舟**模型推理**服务中使用这些豆包模型产生的 Token 用量。

## 抵扣系数

使用豆包大模型时产生的 Token 可以通过特定的抵扣系数转换为资源包点数，购买豆包大模型资源包后，系统自动根据抵扣系数从资源包中豆包大模型的 Token 用量。抵扣系数和资源包用量的换算关系为：

**资源包用量（积分）= Token 用量 ✖️ 抵扣系数**

各个模型的抵扣系数如下。扣子模型和方舟模型的区别可参考[如何区分方舟模型和扣子模型？](https://docs.coze.cn/guides/model_service#97c89470)。

例如对于使用 Doubao-pro-32k-推理服务的智能体，调用 1 次智能体时输入 400 tokens，输出 600 tokens，扣减 1.52 积分，计算方式为：

**400 tokens/1000 × 0.8 （抵扣系数）+ 600 tokens/1000 × 2 （抵扣系数）= 1.52 积分**

## 抵扣规则

购买豆包大模型资源包后，费用结算时系统优先从资源包中抵扣用量。如果用量超出资源包额度，根据按量计费的价格统计资源用量。你可以在按量付费模式下观察 Token 用量，在业务稳定之后合理评估用量，购买合适额度的资源包。

> [!NOTE]
>
> **说明**
>
> 账号中存在多个资源包时，优先抵扣到期时间更早的积分。
