# 知识库常见问题

## 知识库没有召回预期内容如何处理？

如果没有召回预期文本片段，你可以尝试：

- 在知识库页面中，调整知识库的分片方式，提高每个分片的语义完整度，增加召回概率。

- 在低代码智能体的编辑页面中，调整知识库的召回方式，包括自动召回、按需召回，或每次召回的片段个数。

- 在知识库详情页中，手动为知识库片段增加特定的关键词，增加召回概率。

## 知识库没有召回任何内容如何处理？

如果没有召回任何文本片段，你需要在低代码智能体的编辑页面中，检查知识库是否选择了按需召回，而非自动召回。

- 如果选择按需召回，请确认在低代码智能体的人设与提示词中，明确填写了期望召回的时机。

- 你也可以切换为自动召回，每次用户发送消息时，低代码智能体都会搜索知识库并召回相关的文本片段。

## 文本知识库可以召回图片吗？

文本知识库支持召回图片。在文本知识库中，除 .doc 格式以外的文档类型（如.docx）均支持召回图片。如果你需要上传包含图片的 Word 文档，建议采用 .docx 格式。

## 知识库文件是否支持导出？

不支持将知识库文件下载或导出为外部文件。

## 知识库文件 ID 是全局唯一吗？

知识库文件的 document id 是全局唯一的，每个文件在系统中都有一个唯一的标识符，即使在不同的知识库中，document id 也不会重复。

## 知识库文档批量自动更新后提示“系统内部异常”，如何处理？

当你通过**在线数据**方式上传文档到文本知识库，并设置更新频率为按天自动更新时，可能会由于系统资源限制，遇到“系统内部异常”问题。

## 执行工作流时知识库节点报错，如何处理？

执行工作流时知识库节点报错，可能原因是知识库已被删除。请检查工作流中引用的知识库是否已被删除，如果已经删除，需要重新创建知识库，并更新工作流中引用的知识库节点。

## 如何取消第三方平台授权？

当你不需要将飞书、微信、Notion 等平台的文件导入到知识库时，你可以在**账号设置**的**数据源**页签中，将光标悬浮至需要移除授权的账号区域，然后单击**删除**图标。具体操作，请参考[移除授权](https://docs.coze.cn/guides/manage_data_source_permissions)。

## 为什么采集网站图片失败？

采集网站图片失败的可能原因如下：

- 图片 URL 以 http 开头，而不是 https，这不符合浏览器的安全标准，可能导致图片采集失败。

- 在线采集网站图片时，系统会优先从缓存中获取内容。如果缓存中没有命中，才会触发实时抓取。实时抓取通常不会解析网页中的图片或超链接，可能导致图片采集失败。请确保缓存中包含需要的图片资源。

## 为什么选择上传飞书文档到知识库时，显示暂无符合权限的数据？

通常是因为当前知识库未获得足够的权限或飞书文档本身不符合导入要求，你可以参考以下方法解决：

- 如果还未授权，请根据页面提示完成授权。

- 检查飞书文档权限。

- 上传飞书文档到知识库时，支持导入**我的空间**下所有者为本人的表格文档以及**知识库Wiki**中的表格文档，暂不支持导入共享空间下的表格文档。

## 上传包含图片的 Markdown 文件时如何正确存储图片？

当你上传包含图片的 Markdown 文件到知识库时，文件中的图片 URL 或 Base64 编码将保持原格式，无法被识别为图片。建议将带图片内容保存为 .docx 格式文件再上传。

## 上传 Excel 文件到知识库，只展示第一列数据，如何解决？

上传 Excel 文件到知识库时，如果只展示第一列数据，可能是由于文件内容或格式问题导致的。建议你检查 Excel 文件的内容、格式是否正确，并确认没有隐藏列或行。

## 使用知识库时，如何做到分片调取更准确？

你可以参考以下建议，以实现知识库分片调取更准确。

- 保证原始数据的准确性和完整性。知识库中的数据质量和准确性，将直接影响分块调取的效果。

- 将相关的内容组织成一个知识单元，确保每个单元的主题明确，内容相对独立和完整。

- 分片是模型查询的最小单位，确保每个分段内容的完整度，避免将关键信息分割在不同的分段中。

- 在调取信息时，使用清晰、准确的语言描述需求，以便大模型能够理解并返回最相关的内容。

## 扣子编程支持接入本地知识库吗？

扣子编程暂不支持接入本地知识库。你可以开发自定义插件，通过插件方式访问本地知识库或私有化知识库。创建自定义插件的步骤，请参考[操作导航](https://docs.coze.cn/guides/plugin)。

## 分片后的知识库存储空间是怎么计算的？

知识库存储空间是以切片后的文件大小为准。例如本地文件大小为 500MB，上传文件到知识库转化成分片数据后可能是 8000MB，那么占用的知识库空间大小将以 8000MB 为准。

## 为什么上传知识库文件失败，提示错误码 4306？

上传知识库文件失败，提示错误码 4306 是因为文本类型知识库的文件数量已达上限，文本类型知识库最多支持 300 个文件。若需上传更多文件，可创建多个知识库或合并文档。
