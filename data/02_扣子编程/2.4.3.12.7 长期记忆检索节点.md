# 长期记忆检索节点

低代码工作流中的长期记忆检索节点用于在工作流中检索记忆库中存储的用户喜好、用户画像等个性化信息，使模型的输出更贴合用户需求。

## 升级长期记忆功能

> [!NOTE]
>
> **注意**
>
> 升级到新版记忆库后，历史的长期记忆数据将被重置，无法找回。

长期记忆功能已完成全面升级，升级后：

- 系统会将长期记忆统一写入指定的新版记忆库，记忆召回服务的准确性与稳定性显著提升。

- 各记忆库间数据相互隔离，并且系统会通过用户 UID 与渠道 ID 组合生成编码，该编码将作为数据隔离的核心标识，保障同一记忆库中不同主体的记忆数据独立。

- 新版记忆库支持绑定多个低代码智能体、工作流，可实现记忆数据共享。

新建的低代码工作流仅能选择长期记忆新节点（包含写入、检索节点）；包含旧长期记忆节点的存量低代码工作流，你可以将旧节点替换为新节点。

## 节点说明

在用户喜好推荐等个性化的场景中，通常需要基于用户画像、关键记忆点等个人数据进行推荐、筛选，让模型效果更加贴合用户需求、提高用户体验。你可以在工作流的长期记忆检索节点中添加指定的记忆库，从中检索用户喜好、用户画像等信息，并在下游节点中加入大模型相关的节点，让大模型基于检索到的长期记忆进行信息召回，最终返回更符合用户需求的个性化回复。

> [!NOTE]
>
> **说明**
>
> 不支持在低代码应用中使用记忆库功能，即在应用中创建工作流时，不支持添加长期记忆写入节点和长期记忆检索节点。

## 计费说明

在使用长期记忆检索节点从记忆库中检索长期记忆时，将产生检索费用。具体的计费项及单价，请参考[记忆库费用](https://docs.coze.cn/coze_pro/memory_fee)。

## 添加节点

在工作流画布中，单击 **+ 添加节点**，在**知识库&数据**区域选择**长期记忆检索**节点，即可将节点添加到画布中。

## 配置节点

### 记忆库

各记忆库间数据相互隔离，并且系统会通过用户 UID 与渠道 ID 组合生成编码，该编码将作为数据隔离的核心标识，保障同一记忆库中不同主体的记忆数据独立。检索长期记忆时，需先指定目标记忆库，系统将基于用户 UID 与使用渠道匹配对应的编码值进行检索。

在**记忆库**区域，单击 **+**，选择目标记忆库。

<img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/a006f62118c24d6fba8a312e68cc2437~tplv-goo7wpa0wc-quality:q75.awebp" width="661">

### 输入

在**输入**区域，配置检索长期记忆的关键信息。

<img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/5d3a75e2ccb0409f8c1a5039489982f0~tplv-goo7wpa0wc-quality:q75.awebp" width="217">

| **配置** | **说明** |
| --- | --- |
| query | 输入用于从记忆库检索长期记忆的关键信息，例如查询用户的喜好、生日、名字等信息。以新闻搜索的工作流为例，query 可以固定为喜欢什么类型的新闻，基于用户喜好检索并推荐新闻。  query 为必填参数，其值可以引用变量或直接输入。  - 引用：引用变量或上游节点的输出参数。  - 输入：指定为具体的字符串。 |
| limit | 检索的最大记忆数量，必填参数，初始值为 20 条。 |

### 输出

输出参数固定为 outputList，Array<Object> 类型，返回与检索信息相关的长期记忆。在下游节点引用了这个参数后，模型会总结长期记忆中的内容，作为下游节点的输入。
