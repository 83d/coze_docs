# 部署运维概述

部署是将你在扣子编程开发的 AI 编程项目，一键发布为线上服务的过程。它实现了从创意构思到生产可用的端到端落地，让你无需关心服务器配置、域名解析等繁琐的工作，即可将作品交付给最终用户。

## 部署的核心能力

### 多终端产物部署

扣子编程支持将不同类型的项目部署为对应形态的线上服务。各类项目部署后的产物形态与访问方式如下：

| **项目类型** | **部署产物** | **访问方式** |
| --- | --- | --- |
| **网页应用** | 动态网站 | 通过扣子编程提供的默认域名或你绑定的自定义域名，在浏览器中直接访问。 |
| **智能体** | API 服务 | 自动生成 API 服务，供其他应用或服务调用。 |
| **工作流**  | API 服务 | 自动生成 API 服务，供其他应用或服务调用。 |

### 自定义域名

扣子编程支持绑定你自己的独立域名，以增强品牌辨识度。同时，扣子编程也为每个部署的网页应用或 API 服务**自动分配默认域名**（.coze.site 后缀），无需任何额外配置即可立即访问。

### 部署历史与回滚

扣子编程会自动记录每一次的部署版本。当线上版本出现问题时，你可以随时查看历史记录，并一键**回滚**到任意一个可用的历史版本，快速恢复服务。

### 运行日志

你可以直接在部署管理页面查看已发布项目的后端**运行日志**，以便在线上故障排查时，快速定位问题根源。

### 部署审核

部署网页应用时，扣子编程会对网页应用的名称、描述以及网页内容进行审核，拦截包含敏感信息的网页。

## 部署流程

整个部署过程为一条自动化的流水线，你只需进行少量关键配置，即可完成发布。

1. **部署配置**

- 在部署配置页面进行域名、数据库同步、生产环境变量等相关配置。

2. **自动部署**

- 单击**开始部署**后，扣子编程将自动执行部署操作。在整个过程中，你可以实时查看各阶段的**部署日志**。如果遇到错误，日志中会高亮显示相关信息，帮助你快速定位并修复问题。部署流程包括：

- **打包**：将编译产物打包成可部署的格式。

- **构建**：执行构建指令，编译项目代码。

- **部署**：将项目包部署到线上服务器。

3. **部署审核**

- 扣子编程会对 AI 生成的网页应用进行审核，若存在违规内容，会将其封禁。智能体和工作流暂不审核。

4. **发布后验收与消费**

- 部署成功后，你可以立即对项目进行体验和验收。

## 使用限制

项目部署后，生产环境中使用项目的限制说明如下表所示。

| **项目** | **说明** |
| --- | --- |
| API 并发限制 | 智能体和工作流部署为 API 后，调用单个 API 的并发限制 为 200 次/秒。 |
| 上传文件大小限制 | 在项目中上传文件时，文件大小最大为 **16 MB**。  如果项目中经常需要传输较大文件，你可以在 AI 编程开发页面的左侧对话框中，让扣子 AI 实现分片上传。指令示例：   当文件超过 16MB 时会遇到错误，现在帮我实现一个分片上传接口，当文件大小大于 16MB 时，将文件分片上传，单个分片不大于 4MB。  |
| HTTP 连接超时（有数据交互） | 当客户端与服务端持续进行数据传输时：  - 上传文件的超时时间为 **15 分钟**。  - 下载文件的超时时间为 **15 分钟**。 |
| HTTP 连接保持时间（无数据交互） | 当客户端与服务端无任何数据传输时，HTTP 连接最长保持 **90 秒**。  例如，在生图渲染、多轮对话聊天等耗时较长、需要持续连接的场景中，为避免连接断开，需每 90 秒至少发送一次心跳信号，确保服务器与设备的连接持续活跃。你可以在 AI 编程开发页面的左侧对话框中，让扣子 AI 生成相关代码。指令示例：   帮我实现一个心跳保活功能：在生图任务执行期间，每 85 秒自动向服务端发送一次 PING 信号（避免 90 秒超时阈值），确保 HTTP 长连接不中断，包含连接状态监测和自动重连逻辑。  |
