# 低代码工作流使用限制

使用低代码工作流时，应注意以下限制。

## 超时时间

不同低代码工作流节点的超时时间如下：

| **场景** | **超时时间** | | | | | | |  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  | **模型节点、**  **插件节点、语音播客** | **HTTP 节点** | **视频生成** | **图像生成节点** | **数据库节点、意图识别节点、代码节点、画板节点、视频抽帧** | **其他节点** | 低代码**工作流整体** |
| 默认值 | 3 分钟 | 2 分钟 | 6 分钟 | 10 分钟 | 1 分钟 | 无单节点限制 | - 同步运行：10 分钟  - 异步运行：24 小时 |
| 最大值 | 10 分钟 | 10 分钟 | 10 分钟 | 10 分钟 | 1 分钟 | 无单节点限制 |  |

> [!NOTE]
>
> **说明**
>
> - 2025年4月24日之前创建的大模型节点默认超时时间为 10 分钟；IDE 插件为 1 分钟。你可以通过节点中的**异常处理配置**来调整超时时间，详细信息可参考[异常处理](https://docs.coze.cn/guides/llm_node)，注意部分节点不支持手动调整超时时间。
>
> - 在智能体中启用低代码工作流异步运行功能的操作，请参考[设置异步运行](https://docs.coze.cn/guides/agent_workflow)。在低代码工作流中可以通过异步任务及异步触发节点实现低代码工作流的异步运行，具体操作请参考[异步执行低代码工作流](https://docs.coze.cn/guides/execute_workflow_asynchronously)。
>
> - 端插件类型的插件节点无超时时间限制。
>
> - 未开启异步运行时，为确保低代码工作流运行的准确性和稳定性，建议将低代码工作流的执行时间控制在 5 分钟以内。超过 5 分钟，可能因复杂情况影响执行结果的准确性。
>
> - 一次对话中，智能体连续两次回复的时间间隔限制为 10 分钟。

## QPS 限制

执行低代码工作流时，单个节点维度的 QPS 限制和不同订阅套餐在单个低代码工作流维度的 QPS 限制如下表所示。

> [!NOTE]
>
> **说明**
>
> 主账号及其所有子账号共享 QPS 限制。

- 单个节点维度

-

  |  |  |  |
  | --- | --- | --- |
  | **节点** | 数据库相关节点 | 其它节点 |
  | **QPS** | 400 | 3000 |


- 单个工作流维度

-

  |  |  |  |  |  |  |  |
  | --- | --- | --- | --- | --- | --- | --- |
  | **订阅套餐** | 个人免费版 | 个人进阶版 | 个人高阶版 | 个人旗舰版 | 企业标准版 | 企业旗舰版 |
  | **QPS** | 200  | 500 | 500 | 500 | 500 | 500 |


## 其他限制

| **限制** | **说明** |
| --- | --- |
| 运行次数 | 试运行指通过扣子编程的低代码工作流编排页面试运行工作流。按照每次试运行消耗的模型 token 扣减积分。关于积分的更多信息，请参考[积分](https://docs.coze.cn/coze_pro/credits)。   <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/fbd56fbe8d9349f8bb815050603b05fb~tplv-goo7wpa0wc-quality:q75.awebp" width="1136">  |
| 节点数量 | - 每个低代码工作流中最多添加 1000 个节点。每次运行低代码工作流时，包含循环节点的循环次数、子工作流的运行节点在内，每次最多执行 1000 个节点。  - 每个低代码工作流中最多添加 50 个代码节点。 |
| 节点并发 | - 扣子主账号及其所有子账号共享图像插件节点并发限制，具体的并发限制请参考[插件费用](https://docs.coze.cn/coze_pro/plugin_fee)。  - 代码节点的并发限制为每个工作空间内 500。 |
| 请求大小 | 低代码工作流支持的请求大小上限为 20MB，包括输入参数以及运行期间产生的消息历史等所有相关数据。 |
| 节点入参 | - 运行低代码工作流时，工作流节点的输入、输出的上限均为 10MB；代码节点、插件节点入参上限为 2MB，超出限制会报错 Invalid Request: input size exceeded，或使用省略号截断。  - 试运行 IDE 插件节点时，其输出上限为 64KB。 |
| 用户变量长度 | 用户变量存储最大支持约 6 万字符，出于性能考虑不建议存储过长的内容。 |
