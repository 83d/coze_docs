### OpenClaw 部署体验优化

扣子编程中部署 OpenClaw 现已支持以下能力：

### 2026 年 02 月 06 日

### OpenClaw 个人助理

扣子编程现已支持云端环境快速部署 OpenClaw 个人助理，只需简单的配置，即可拥有一个 7\*24 小时不间断运行的个人助理。和传统的 AI 助理不同，OpenClaw 不是普通的聊天机器人，被授予操作权限之后，它能控制你的终端执行操作，可以“动手工作”，而不是“只出主意”。

更多信息，请参考[一键部署 OpenClaw 并集成飞书](https://docs.coze.cn/tutorial/openclaw)。

## 2026 年 01 月 28 日

### 图像生成节点

图像生成节点新增 Seedream 4.5 模型。相较于 Seedream 4.0，Seedream 4.5 模型有了整体提升，显著改善了人像效果和画面美感，并增强了推理能力，详情请参考[图像生成节点](https://docs.coze.cn/guides/image_generation_node)。

## 2026 年 01 月 19 日

### 数据向量化

扣子编程支持调用向量模型，将非结构化内容转换为向量并存储于数据库中，提升系统对非结构数据的召回能力。更多信息，请参考[数据向量化写入与检索](https://docs.coze.cn/guides/vector_based_data_writing_and_search)。

### 内置集成

扣子编程集成了向量模型，将文本、图像、视频等数据转换为数值向量。更多信息，请参考[内置集成](https://docs.coze.cn/guides/internal_integrations)。

## 2026 年 01 月 15 日

### 内置集成

内置集成服务新增视频生成与内容处理能力。更多信息，请参考[内置集成](https://docs.coze.cn/guides/internal_integrations)。

- 视频生成：AI 编程项目支持接入 Doubao-Seedance-1.5-pro 大模型，生成视频。

- 内容处理：AI 编程项目支持接入内容处理工具，剪辑视频。

## 2025 年 12 月 17 日

即日起，「扣子开发平台」现已正式升级为「**[扣子编程](code.coze.cn)**」！本次升级也标志着**扣子编程**将从低代码 AI Agent 开发平台转变为基于 AI 编程的全代码应用开发平台，提供更低的操作门槛、更高的开发上限，将你的精力聚焦到创意本身。

关于扣子编程的产品介绍，可参考[什么是扣子编程](https://docs.coze.cn/guides/welcome)。

### AI 编程能力

AI 编程是**扣子编程**的核心能力，你只需要清晰表达意图，就能一站式完成从**构建到落地**的完整生产级闭环，覆盖**智能体**、**工作流**、**网页应用**等多个开发场景。

在浏览器中打开**[扣子编程](code.coze.cn)**，只要清晰描述你的需求并敲击回车，即可开始开发你的第一个 AI 编程项目，让你的开发过程更聚焦于创意和价值创造。

关于 AI 编程能力的详细说明，可参考[扣子 AI 编程概述](https://docs.coze.cn/guides/vibe_coding_overview)。

### 集成服务与生态

扣子编程将复杂的能力配置包装为独立的集成服务模块。在 AI 编程时，只需清楚描述你的需求，扣子编程会自动判断并接入所需的集成服务：

- **内置集成**：内置如数据库、存储、身份认证等集成服务，开发者无需手动配置。

- **外部集成**：由扣子编程官方提供的第三方集成服务，包括飞书、微信、火山方舟等服务。

关于集成服务的详细说明，可参考[集成服务概述](https://docs.coze.cn/guides/integrations_overview)。

### 一键发布能力

扣子编程为你的 AI 编程项目提供一键发布功能：

- 将你的智能体和工作流一键发布为 API 服务，以便集成到现有系统中。

- 将你开发的 AI 应用发布为独立的网页站点，支持自定义域名。

只需要点击“部署”，扣子编程会自动完成云端打包、构建与服务部署。

关于部署能力的详细介绍，可参考[部署运维概述](https://docs.coze.cn/guides/deployment_ops_overview)。

### AI 生成式应用功能升级

AI 生成式应用的功能现已全面升级，支持开发具备完整前后端逻辑的应用，并将其部署上线。原入口已屏蔽，你可以前往新版**[扣子编程](code.coze.cn)**页面体验，详细操作步骤请参考[开发网页应用](https://docs.coze.cn/guides/vibe_coding_web_app)。

## 2025年11月28日

### MCP 插件

扣子开发平台内置了 MCP 客户端能力，允许你基于已有的 MCP 服务创建自定义插件，轻松、高效地将外部的 MCP 工具能力集成到扣子开发平台。详情请参考[基于 MCP 服务创建插件](https://docs.coze.cn/guides/create_a_plugin_based_on_mcp)。

## 2025年11月25日

### Responses API 模型

部分模型支持将模型协议切换为 Responses API，你可以通过**模型设置**页面或[创建智能体](https://docs.coze.cn/developer_guides/create_bot)、[更新智能体](https://docs.coze.cn/developer_guides/update_bot) API 进行相关配置。Responses API 不仅延续了 Chat API 的易用性，还原生支持高效的上下文管理和**前缀缓存**。详情请参考[切换 Responses API](https://docs.coze.cn/guides/llm)、[扣子模型](https://docs.coze.cn/guides/model_service)。切换为 Responses API 后，扣子侧调用发起对话等 Open API 的方式与 Chat API 一致。

### 异步执行工作流

扣子开发平台支持通过异步任务执行工作流。你可以创建异步任务，并通过任务中心、工作流异步任务节点方式触发执行，以实现工作流的异步执行。详情请参考[异步执行低代码工作流](https://docs.coze.cn/guides/execute_workflow_asynchronously)、[异步任务节点](https://docs.coze.cn/guides/asynchronous_task_node)。

## 2025年11月21日

### 豆包语音合成 2.0 音色

扣子开发平台支持豆包语音合成 2.0 音色，你可以在[语音合成](https://docs.coze.cn/developer_guides/text_to_speech) API 或 WebSocket 语音通话、RTC 视频通话中，使用豆包语音合成 2.0 音色，该音色支持通过 context\_texts 参数控制合成语音的整体情绪（如悲伤、生气）、方言（如四川话、北京话）、语气（如撒娇、暧昧、吵架、夹子音）、语速（快慢）及音调（高低）等。详情配置请参考 [语音合成](https://docs.coze.cn/developer_guides/text_to_speech)、[Realtime 上行事件](https://docs.coze.cn/developer_guides/signaling_uplink_event)、[双向流式对话上行事件](https://docs.coze.cn/developer_guides/streaming_chat_event)、[双向流式语音合成事件](https://docs.coze.cn/developer_guides/tts_event)、[系统音色列表](https://docs.coze.cn/dev_how_to_guides/sys_voice)。

## 2025年11月13日

### 模型定位

在模型下架场景中，你可以通过**用量详情**图表，快速定位调用了待下架模型的智能体或工作流，并为它们替换模型。建议结合 tokens 消耗数据排序，优先替换高频高消耗智能体或工作流中的模型。详情请参考[如何快速替换模型？](https://docs.coze.cn/guides/viewing_model)。

### 智能体发布企业商店

扣子**企业版**支持将智能体发布至企业商店。企业商店是面向企业版用户的专属商店入口与发布渠道，用于在企业范围内上架与分发企业内部的智能体和应用。企业商店中的智能体和应用仅本企业的员工和访客可见、可访问，非本企业成员无法浏览或使用。详情请参考[发布到企业商店](https://docs.coze.cn/guides/publish_agent_enterprise_store)。

### Doubao 视频生成插件

扣子官方提供两款视频生成插件，[Doubao-视频生成 Pro](https://www.coze.cn/store/plugin/7519125835639685155?from=plugin_card) 和 [Doubao-视频生成 Lite](https://www.coze.cn/store/plugin/7519094063849766927?from=plugin_card)。支持基于文本提示词生成视频，也支持基于文本提示词、视频首帧、尾帧图片或参考图共同生成视频。详情请参考[视频生成插件](https://docs.coze.cn/guides/doubao_seedance_plugin)。

## 2025年10月21日

### 导入与导出工作流

导入与导出工作流功能升级，新增如下能力：

- 企业版支持选择高级模式，可同步导出子工作流。当跨空间重新导入后，子工作流可正常运行。

- 导出工作流到本地后，支持在本地编辑工作流文件。

## 2025年10月20日

### 多组织

扣子**企业版**新增支持多组织管理。组织是企业或团队下的虚拟分组，其中包含多个工作空间，便于集中管理不同工作空间下的成员、资源及权限。在大中型企业团队中，可根据部门或项目维度划分组织，实现多业务部门的独立运作与统一管理。详情请参考[了解企业组织](https://docs.coze.cn/guides/team_and_enterprise_overview)、[创建组织](https://docs.coze.cn/guides/manage_organization)。

## 2025年10月16日

### 批量任务

扣子开发平台新增工作流批量执行功能，当你创建批量任务并上传 CSV 格式的工作流输入参数后，系统将托管式批量执行工作流，详情请参考[批量执行低代码工作流](https://docs.coze.cn/guides/batch_run_workflow)。

## 2025年10月15日

### 记忆库

为进一步优化服务体验，提供更强大、更灵活的长期记忆功能，扣子开发平台对长期记忆功能进行了升级。升级后：

- 系统会将长期记忆统一写入指定的新版记忆库，记忆召回服务的准确性与稳定性显著提升。

- 各记忆库间数据相互隔离，并且系统会通过用户 UID 与渠道 ID 组合生成编码，该编码将作为数据隔离的核心标识，保障同一记忆库中不同使用主体的记忆数据独立。

- 新版记忆库支持绑定多个智能体、工作流，可实现记忆数据共享。

详情请参考[记忆库](https://docs.coze.cn/guides/long_term_memory)。

## 2025年9月29日

### 插件 MCP 调用

扣子开发平台支持将扣子官方付费插件和三方付费插件封装为 MCP（Model Context Protocol）工具，便于开发者在支持 MCP Server 的平台（如 Trae、Cursor、Claude 等）轻松调用扣子开发平台插件，详情请参考[通过 MCP 方式调用付费插件](https://docs.coze.cn/guides/call_plugin_mcp)。

## 2025年9月25日

### 自定义模型

新增**接入自定义模型**功能。企业可将自部署或第三方在线模型集成至扣子平台，进一步拓展扣子平台可用的模型范围。详情请参考[接入自定义模型](https://docs.coze.cn/guides/deploy_custom_model)。

扣子开发平台支持**零代码快速生成扣子应用**，开发者无需掌握代码技能，仅需通过自然语言描述应用创意，即可输出可运行的 App 与 Web 端应用，**高效实现创意到落地产品的转化**。

### 公共渠道

支持将智能体发布至中兴、荣耀、联想公共渠道，详情请参考[发布到中兴智能体商店](https://docs.coze.cn/guides/zhongxing_app_store)、[发布到荣耀智能体平台](https://docs.coze.cn/guides/rongyao_app_store)、[发布到联想开放平台](https://docs.coze.cn/guides/lianxiang_app_store)。

## 2025年9月19日

### 侧边栏改版

扣子开发平台对侧边栏进行了改版优化，主要优化导航层级，缩短用户使用路径，主要变更如下：

- **工作空间管理**：调整至一级导航栏，取消原嵌套层级。

- **二级导航栏**：位置从左侧侧边栏迁移至顶部区域。

- **商店功能拆分**：原**商店**模块拆分为独立的**插件商店**与**作品社区**。

- **收藏夹迁移**：从左侧导航栏移除，整合至**个人主页**中。

- **功能入口调整**：通知、账单、产品矩阵的位置迁移至页面左下角。
