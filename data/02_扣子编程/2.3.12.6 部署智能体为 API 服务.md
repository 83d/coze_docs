# 部署智能体为 API 服务

本文介绍如何将你在扣子编程上开发的智能体，快速部署成为可访问的 API 服务，以便后续通过 OpenAPI 方式将智能体的 AI 功能集成到你的应用中。

## 使用限制

- 仅智能体的**所有者**有权限执行部署操作。

- 每个项目最多能创建 10 个 API Token。

## 准备工作

在开始部署之前，请确保你已通过扣子编程开发智能体，且试运行通过。具体可参考[开发智能体](https://docs.coze.cn/guides/vibe_coding_agent)。

## 部署智能体

1. 在[扣子编程](https://code.coze.cn/home)左侧导航栏选择**项目管理**，筛选带有 **New** 标签的项目，单击目标项目。

-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/cf3f91f97b1e4d9094d25b7627b5ce50~tplv-goo7wpa0wc-quality:q75.awebp" width="500">


2. 在 AI 编程开发界面的右上角，单击**部署**。

-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/c2405ecf544549a08e4e23ac39698c60~tplv-goo7wpa0wc-quality:q75.awebp" width="500">


3. 配置数据库同步。

- 生产环境和开发环境的数据隔离。首次部署时，你可以设置是否将开发环境的数据同步至生产环境。后续部署时，扣子编程仅同步开发环境的表结构变更，不同步具体数据。

-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/a4d2eef2c8084c60ab6673e6d7a4a6ee~tplv-goo7wpa0wc-quality:q75.awebp" width="500">


- **关闭**：仅同步开发环境中的数据表结构，不同步数据。

- **开启**：将开发环境中的数据表结构和数据完全同步到生产环境。

4. 配置生产环境变量。

- 在生产环境中为项目配置特定的环境变量，例如 API Key、数据库连接字符串、飞书文档地址等敏感信息，以免将这些信息硬编码在代码中导致安全风险。

- 你可以新建环境变量、查看本次部署新增和变更的环境变量。关于环境变量的具体说明请参考[管理环境变量](https://docs.coze.cn/guides/environment_variables)。

- > [!NOTE]
  >
  > **说明**
  >
  > 部署时新建的变量，仅在生产环境生效，不会被添加至开发环境。


5. 如需查看服务器资源、构建指令、端口等高级部署配置，你可以展开**更多配置**，查看配置详情。

6. 单击**开始部署**。

- 扣子编程将自动进行部署相关操作。你可以在部署页面查看部署的进展和部署日志。部署过程可能需要几分钟，请耐心等待。部署过程中，你可以随时取消部署。

## 获取 API 访问信息

部署成功后，扣子编程会自动生成 API 服务，你可以在**部署总览**页面获取访问 API 的相关信息。

### 查看 API 访问信息

获取该服务的 API 访问地址、请求Header、请求参数等详细信息。

1. 在部署的**总览**页面，单击某条部署记录右侧的更多按钮，选择**查看**。

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/eef320d3063c4f90a0bd05f32110ba72~tplv-goo7wpa0wc-quality:q75.awebp" width="500">


2. 在**API 请求示例及接口说明**页面，查看该服务的 API 访问信息。

-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/52d94dd609844ba98b4e7b9a4429f929~tplv-goo7wpa0wc-quality:q75.awebp" width="500">


### 创建 API Token

你需要创建 API Token，调用 API 时需要使用 API Token 进行身份验证。你需要将创建的 API Token 包含在请求头的 Authorization 参数中。

1. 在智能体开发页面，在右侧单击➕打开新的标签页，在弹出的标签页中选择**部署**。

2. 在部署的**总览**页面，单击某条部署记录右侧的更多按钮，选择**查看**。

3. 在**API 请求示例及接口说明**页面，单击**管理 API Token**，单击**创建 API Token** 生成新的 API Token。复制并妥善保存 API Token。

- > [!NOTE]
  >
  > **说明**
  >
  > - 生成的令牌仅在此时展示一次，请即刻复制并妥善保存。
  >
  > - 请妥善保存该 API Token，不要在浏览器或其他客户端代码中暴露 API Token。
  >
  > - 每个项目最多能创建 10 个 API Token，API Token 的有效期为永久有效。
  >
  > - 暂时不支持在部署详情页面直接调用和调试 API。


-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/b009fab8cedb4c1a8b1849a0b4525778~tplv-goo7wpa0wc-quality:q75.awebp" width="500">


4. （可选）你也可以在 **API Token** 页面查看已创建的 API Token 列表，删除不再使用的 API Token。

## 分享智能体

在智能体开发页面右上角单击**分享**按钮，即可获取智能体的 API 请求示例并分享给其他用户。你需要将创建的 API Token 提供给对方，对方调用时需将该 API Token 包含在请求头的Authorization参数中。

## 调用智能体 API

部署完成后，你可以在你的应用程序或网页中通过 HTTP 请求来调用智能体的 API，以便集成智能体的 AI 能力。

**接口说明**

调用智能体的 API 请求地址的格式为 https://<your\_domain>/stream\_run，是一个流式响应 API。调用该 API 时，服务端不会一次性发送所有数据，而是以数据流的形式逐条发送数据给客户端，数据流中包含智能体执行过程中触发的各种事件，直至处理完毕或处理中断。

API 的请求参数由 AI 编程自动生成，具体参数说明可在部署详情页的 **API 请求示例及接口说明**页面查看。

> [!NOTE]
>
> **说明**
>
> AI 编程项目中不兼容低代码 API 文档中的[上传文件](https://docs.coze.cn/developer_guides/upload_files)等 API 。

**调用方法**

1. 复制扣子编程提供的 Curl 请求命令。

-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/0c99f0481c2d44edadc4a2f961d9b111~tplv-goo7wpa0wc-quality:q75.awebp" width="600">


2. 将 header 中的 <YOUR\_TOKEN> 替换为你在[创建 API Token](https://docs.coze.cn/guides/deploy_agent_as_api_service)中获取的 API Token。

3. 通过 Postman 或相关工具调用对应的 API 。
