# 发布常见问题

## 智能体发布到哪些渠道需要审核？

智能体首次发布到**抖音小程序**、**微信小程序**、**微信订阅号**、**微信客服**、**飞书多维表格（**发布范围为**自己公司可用**或**其他范围****）**时需要经过审核。

审核通常在在 1~7 个工作日内完成，你可以通过发布历史查看审核结果，或在发布页面查看发布结果。

## 智能体发布后的审核需要多久？

当前审核为人工处理，周期为 1~7 个工作日。

## 为什么抖音企业号发布渠道无法使用了？

由于抖音开发平台的企业号策略调整，扣子抖音企业号发布渠道已于**2025年3月13日**正式下线。

## 如何处理应用发布时提示资源打包失败？

当你在应用发布过程中遇到**资源打包失败**或**试运行失败**的提示，但工作流中并未发现相关资源时，你可以尝试以下步骤来解决问题：

1. 使用无痕浏览器重新操作。

- 使用无痕浏览器分别重新试运行工作流和用户界面，确保功能正常后，再次尝试发布。无痕浏览器可以清除缓存和临时文件，避免因缓存问题导致发布失败。

2. 检查应用绑定关系。

- 检查相关应用是否在工作流中残留了绑定组件。如果存在残留绑定关系，需要在应用界面删除旧的绑定关系，然后重新绑定组件，避免因残留数据导致发布异常。

## 如何实现智能体仅指定人员使用，且不公开内部设置？

你可以通过如下两种方法实现智能体仅指定人员使用：

- 通过飞书账号限制 ：将智能体发布到飞书，使用人员通过飞书账号加入指定组织。限制只有组织成员能够访问和使用该智能体，从而精准限定使用人员范围。

- 通过 API 接入限制 ：将智能体发布为 API，调用该 API 将智能体集成至自有项目。你可以在项目中自行实现访问控制，例如访问令牌、用户身份鉴权等。

## 如何发布智能体为模板？

目前暂不支持直接发布智能体为模板，仅支持发布应用为模板，具体请参见[发布为模板](https://docs.coze.cn/guides/publish_to_template)。你可以加入[扣子交流群](https://applink.larkoffice.com/client/chat/chatter/add_by_link?link_token=181m2d44-cc33-48c6-9381-e3b1f881dd53)，关注群内定期开展的模板征集活动。此外，你还可以将作品提交至[模板需求反馈渠道](https://bytedance.larkoffice.com/share/base/form/shrcnuEI99wNNUxNtVb37ehFKOe)，若作品符合上架要求，扣子工作人员会主动与你联系。

## 应用打包失败

**问题现象**

打包时提示以下资源打包失败，请检查后再试。

**排查思路**

如果应用打包失败，单击**查看详情**，根据错误列表中的错误提示检查并修改，重新试运行工作流。

<img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/896ba693a58b420dab201457125534c3~tplv-goo7wpa0wc-quality:q75.awebp" width="700">

打包失败的常见原因如下：

1. **资源类问题**

- 检查应用中引用的各资源的有效性，包括插件和工作流等：

- **检查插件**：检查应用是否引用了已删除的插件，或跨空间引用了无权访问的插件。移除无效插件或跨空间插件，重新添加可用的有效插件。

- **检查工作流**：检查应用所有组件及事件配置中绑定的工作流是否存在，重新绑定有效工作流。

2. **工作流配置类问题**

- **检查代码节点**：试运行工作流，确保每个**代码**节点均能执行成功。

- **检查字段有效性**：若工作流中引用了子工作流，请检查子工作流中的变量、参数等字段是否已失效，修改后重新引用子工作流；若未引用子工作流，逐个检查并修复失效的变量、参数等字段。

- **检查大模型节点**：检查大模型节点中的模型是否已下线，若已下线，替换为官网可用的大模型后重新发布。

- **检查版本号冲突**：若试运行工作流时提示工作流的版本号冲突，参考[低代码工作流报错“工作流的版本号冲突: xxx” ，应如何处理？](https://docs.coze.cn/guides/workflow_faq)，排查并统一子工作流版本。

- **检查 SQL 语法错误**：检查数据库节点中 SQL 语句是否包含无法识别的字段或语法错误。

- **检查与 MySQL 保留字冲突**：检查数据库节点中，是否使用了数据库关键词或保留字（例如 JSON\_TABLE）作为表字段名称，具体请参考[扣子数据库关键词和保留字有哪些？](https://docs.coze.cn/guides/database_faq)

## 如何获取加盖鲜章的备案合同？

智能体或扣子应用上架小程序、应用商店等渠道时，如果审核方提示需要算法备案相关材料，你可以在火山引擎控制台的[合同管理](https://console.volcengine.com/finance/contract/)页面，获取上架的算法备案相关材料，包括加盖鲜章的备案合同，在备案合同中会提供算法备案号。详细步骤可参考[客户应用上架指南-算法备案资质申请流程](https://www.volcengine.com/docs/82379/1326340)。

申请备案合同需满足以下条件：

- 账号存在方舟大模型或火山引擎豆包系列相关付费订单。

- 账号存在对应商品的预付费/部分预付实例，或运行中的后付费实例，且近 6 个月的调用量大于 0。

- 后付费账单在次月第 4 个自然日后可

> [!NOTE]
>
> **说明**
>
> - 若使用 DeepSeek 等第三方模型需自行申请算法备案，具体可参考[大模型备案说明(含算法备案及人工智能备案)](https://www.volcengine.com/docs/82379/1471389)。
>
> - 若使用火山引擎豆包系列模型，可通过申请豆包合作协议及备案证明获取材料，详细步骤可参考[客户应用上架指南-算法备案资质申请流程](https://www.volcengine.com/docs/82379/1326340)。

### 为什么智能体发布自定义渠道后，对话时提示 200 错误？

智能体发布喵伴等自定义渠道时，需要将智能体发布至 **API 和自定义渠道**，发布 API 的具体操作请参见[发布为 API 服务](https://docs.coze.cn/guides/publish_agent_api)。
