# 设置空间成员角色

在空间的**设置**页面，所有者或管理员可以为其他成员设置角色。

## 空间成员角色权限

不同角色的权限说明如下，其中，✔️表示当前角色有权限，-表示没有权限。

> [!NOTE]
>
> **说明**
>
> 空间资源指空间中的智能体、应用、插件、工作流、知识库、卡片、提示词等资源。

| **权限分类** | **权限点** | **说明** | **所有者** | **管理员** | **成员** |
| --- | --- | --- | --- | --- | --- |
| 管理空间 | 添加成员 | 直接搜索用户，将其添加到空间中。  打开邀请链接，邀请用户加入空间，用户访问链接时可以选择是否加入此空间。链接分享为开启状态时，任何人都可以通过分享链接加入空间。 | ✔️ | ✔️ | - |
|  | 移除成员 | 从空间中移除某个用户。 | ✔️ | ✔️ | - |
|  | 调整角色 | 按需为空间中的用户设置空间角色，可以设置为成员或管理员。 | ✔️ | ✔️ | - |
|  | 删除空间、修改空间名称、转让空间所有权 | 空间一旦删除无法恢复，空间内的所有资源和数据也会同步删除。 | ✔️ | - | - |
|  | 离开空间 | 普通成员和管理员可以随时离开空间，所有者转移空间所有权后才能离开空间。  离开空间后，用户创建的资源会转移给空间所有者，这些资源的协作者权限不变。 | ✔️ | ✔️ | ✔️ |
| 管理资源 | 创建、查看、复制 | 在空间中创建、查看、复制智能体等资源。 | ✔️ | ✔️ | ✔️\*① |
|  | 修改、发布 | 默认仅项目的所有者可修改、发布项目。 | \*③、④ | | |  |  |
|  | 导入 | 目前仅低代码工作流支持导入功能。 | ✔️ | ✔️ | ✔️ |
|  | 导出 | 目前仅低代码工作流支持导出功能。  - 低代码工作流的所有者、协作者、空间所有者或管理员可以导出工作流。  - 空间成员不可导出其他成员的低代码工作流。 | ✔️ | ✔️ | - |
|  | 删除资源 | 空间成员不可删除其他成员拥有的资源。 | ✔️ | ✔️ | - |
| 文件夹 | 创建、编辑、删除 | 在工作空间中创建、编辑、删除文件夹。 | ✔️ | ✔️ | -\*② |
|  | 资源移动 | 将资源移入文件夹，或从文件夹移出。 | ✔️ | ✔️ | 仅资源所有者可以移动资源。 |

> [!NOTE]
>
> **说明**
>
> \*①：工作空间成员默认可以创建和复制智能体、应用、知识库、数据库、插件，空间所有者和管理员也可以禁止空间成员创建这类资源。工作流、音色和卡片不受此限制，具体请参见[设置空间成员权限](https://docs.coze.cn/guides/publish_permission)。
>
> \*②：工作空间成员默认没有权限创建和编辑文件夹，空间所有者和管理员也可以允许空间内成员创建和编辑文件夹，具体步骤请参见[新建、编辑文件夹权限](https://docs.coze.cn/guides/publish_permission)。
>
> \*③：AI 编程开发的智能体、应用和工作流不支持多人协作，仅企业标准版和企业旗舰版中的**低代码**智能体、应用和工作流支持将工作空间中其他成员设置为协作者。低代码智能体和工作流的协作者可以协同编辑、发布智能体或工作流。
>
> \*④：插件（不包括基于 IDE 创建的插件）的所有者、空间所有者或管理员可以修改、发布插件。其他资源仅资源所有者或协作者可以修改、发布资源。

## 调整成员角色

空间所有者和管理员可以调整空间中任意成员的角色，支持调整为空间成员或管理员。

1. 登录[扣子编程](https://code.coze.cn/home)。

2. 在左下角单击个人头像，选择企业，然后单击对应组织的**设置**图标。

-

  <img src="https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/e1eb91083b2e44cd8ffccc9e79e677e3~tplv-goo7wpa0wc-quality:q75.awebp" width="311">


3. 在**组织管理**页面的顶部选择**空间管理**，然后单击目标工作空间对应的**设置**。

3. 在顶部选择**成员管理**页签，找到指定成员，并修改角色。
