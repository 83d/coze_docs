# 数据库常见问题

## 低代码智能体或应用中的数据库数量有限制吗？

- 在低代码应用中添加数据库时，数据库数量无限制。

- 在低代码智能体中添加数据库时，最多可添加 3 个数据表。

详细说明，请参考[使用限制](https://docs.coze.cn/guides/database_overview)。

## 什么是扣子数据库测试数据和线上数据？

默认情况下，扣子数据库的测试数据和线上数据按下述规则分类。同时，你也可以调整各个发布渠道的数据读写目标，切换渠道访问的是线上数据还是测试数据，详情请参考[设置渠道读写配置](https://docs.coze.cn/guides/manage_database_resources)。

- **测试数据：**开发者在扣子编程中调试低代码智能体或应用时产生的数据。

- **线上数据：**低代码智能体或应用发布到抖音、豆包、飞书等平台（默认情况下，不包括扣子商店、模板）后，在这些平台上实际运行时产生的数据。

## 扣子数据库测试数据的隔离规则是什么？

同一工作空间下，无论是低代码智能体还是工作流，只要引用了相同的数据库，就能共享该数据库中的测试数据。

例如，在工作空间 A下创建了智能体 A 和工作流 B，并且智能体 A 和工作流 B 使用了同一个数据库，那么智能体 A 产生的测试数据，在工作流 B 也可查看。

> [!NOTE]
>
> **说明**
>
> 如果工作流数据库节点添加的是应用内创建的数据库，而非资源库中的数据库，那么试运行工作流产生的数据仅限在该应用内查看。

## 扣子数据库线上数据的隔离规则是什么？

线上数据支持按照发布渠道隔离数据，其渠道隔离规则以你创建数据库时选择的渠道模式为准。例如选择**渠道隔离**，则表示用户在一个渠道（如飞书）中产生的数据无法在另一个渠道（如微信）查看。

## 如何查看扣子数据库中的测试数据？

你可以在数据库详情页的**测试数据**页签中，查看数据库的测试数据，详情请参考[查看表数据](https://docs.coze.cn/guides/manage_data_sheets)。

## 如何查看扣子数据库线上数据？

你可以在数据库详情页的**线上数据**页签中，查看数据库的线上数据，详情请参考[查看表数据](https://docs.coze.cn/guides/manage_data_sheets)。

## 扣子数据库的测试数据和线上数据的权限规则是什么？

测试数据和线上数据的权限规则可能存在差异，具体如下：

- 测试数据：数据库所在工作空间的所有成员都能新增、查看、修改及删除空间内数据库的测试数据。

- 线上数据：数据库的创建者能新增、查看、修改、删除该数据库的线上数据，而使用者只能查看空间内数据库的线上数据，不能新增、修改或删除。

> [!NOTE]
>
> **说明**
>
> - 数据库的创建者是指创建数据库的开发者，而数据库的使用者是指那些非创建者但需要使用数据库的其他开发者，例如低代码智能体或应用的协作者。
>
> - 发布到商店、模板的线上数据默认存储在测试数据中，遵循测试数据的权限规则；发布到除商店以外的渠道的数据默认存储在线上数据中，遵循线上数据的权限规则。

## 为什么低代码智能体调试时能获取扣子数据库数据，但在 API Playground 中无法获取同样的结果？

默认情况下，智能体的调试环境和 API Playground 的数据源是隔离的。智能体调试时访问的是测试数据，而调用 API 时访问的是线上数据。若需验证 API 渠道数据，你可以通过如下两种方式配置数据，再进行验证。

- 方式一：通过 API 重新添加数据库的线上数据。

- 方式二：修改 API 渠道的读写配置，将 API 渠道的数据读写目标修改为测试数据，以实现调用 API 也访问测试数据 。具体操作，请参考[设置渠道读写配置](https://docs.coze.cn/guides/manage_database_resources)。

## 扣子编程支持接入本地数据库吗？

扣子编程暂不支持接入本地数据库。你可以开发自定义插件，通过插件方式访问本地数据库。创建自定义插件的步骤，请参考[操作导航](https://docs.coze.cn/guides/plugin)。

## 是否支持导出数据库？

支持将扣子数据库及数据库表导出为本地文件。具体操作，请参考[导出表数据](https://docs.coze.cn/guides/manage_data_sheets)。

## 如何备份数据库？

你可以通过如下方式备份数据库。

- 导出数据库表到本地，具体操作，请参考[导出表数据](https://docs.coze.cn/guides/manage_data_sheets)。

- 在工作流中添加**查询数据节点**，全量查询数据库中的数据，并通过**插件节点（****[飞书多维表格](https://www.coze.cn/store/plugin/7395043460165779483?from=store_search_suggestion)****）**将查询结果写入到飞书多维表格中，以实现数据库备份。如何添加节点，请参考[查询数据节点](https://docs.coze.cn/guides/database_select_node)、[插件节点](https://docs.coze.cn/guides/plugin_node)。

## 使用扣子数据库时遇到保留字段报错，如何处理？

使用数据库时遇到保留字段报错，可按照以下步骤处理：

1. 检查表字段名称。

- 确认是否使用了数据库关键字或保留字（例如 JSON\_TABLE）作为表字段名称。如果是，需修改表字段名称。扣子数据库不支持使用数据库关键词或保留字作为数据库表字段名称，详情请参考[扣子数据库关键词和保留字有哪些？](https://docs.coze.cn/guides/database_faq)。

2. 检查 SQL 语法。

- 确认 SQL 语法配置正确。

## 火山数据库关键词和保留字有哪些？

在设计数据库（如定义表名或字段名）时，请避开以下保留字。

- 扣子编程系统内置字段：uuid、sys\_platform、bstudio\_create\_time

- 通用保留字：火山数据库还有一些关键词和保留字，具体列表，请参考[禁用词列表](https://www.volcengine.com/docs/6313/66162?lang=zh)。
