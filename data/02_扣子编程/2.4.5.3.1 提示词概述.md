# 提示词概述

提示词是一种自然语言指令，它为大语言模型（LLM）提供任务指导。搭建低代码智能体的第一步就是编写提示词，为低代码智能体设定身份和目标。低代码智能体会根据大语言模型对人物设定和回复逻辑的理解，来响应用户问题。因此提示词编写得越清晰明确，低代码智能体的回复也会越符合预期。

## 功能特性

- **AI 生成**：你可以通过自然语言告诉 AI 你希望编写或优化的提示词，大语言模型会根据你的描述，自动生成提示词；你也可以根据调试结果，告诉 AI 提示词哪里不符合预期以及你的预期效果，大语言模型会自动帮你完成优化。

- **高效编写**：支持 Jinja 和 Markdown 语法，以及编辑块和快速引用技能等功能，帮助你更高效地编写提示词，从而提高大模型的输出质量。

- **模板复用**：扣子编程提供了各种业务场景的提示词模板，你可以直接使用这些模板，也可以将其修改为更适合自己业务的提示词。

- **资源管理：**你可以在资源库中创建提示词，供工作空间内的其他成员引用。通过更新和丰富提示词资源库，可以不断提高提示词的编写效率。

具体使用说明，请参考[设置提示词](https://docs.coze.cn/guides/set_prompt)、[编写提示词](https://docs.coze.cn/guides/write_prompt)。

## 提示词类型

提示词包括系统提示词和用户提示词，两者区别如下：

- **系统提示词**：在搭建低代码智能体时，设置的**人设与回复逻辑**就是系统提示词，它是开发者为大语言模型设定的初始参数和行为准则，在整个会话中持续影响大模型的响应模式。通过编写系统提示词，可以为大模型设定特定的角色定位和回复逻辑。

- **用户提示词**：在与低代码智能体对话时，用户的输入就是用户提示词，它是用户直接向大语言模型提出的具体指令或问题，指导模型执行特定的任务或提供特定的信息。用户提示词的设置应简洁明了，以便模型能够准确捕捉用户的需求。

假设我们需要构建一个友好且专业的健康咨询助手，以下是设置系统提示词和用户提示词的示例：

- 系统提示词：你是一个友好且专业的健康咨询助手，专注于为用户提供基于科学和医学知识的健康建议。在回答用户的问题时，你的回答应该既专业又易于理解，同时保持语言的温和和鼓励性。请确保所有建议都是基于最新的健康指南，并避免提供具体的医疗诊断。

- 用户提示词：我最近总是感到疲劳，这可能是什么原因呢？。

## 权限说明

提示词不支持多人协作。提示词的所有者可以编辑和删除自己创建的提示词，工作空间所有者和管理员可以编辑和删除工作空间内其他成员创建的提示词，但工作空间内的普通成员没有此权限。

角色不同，提示词的操作权限也不同，详细差异对比如下：

| **操作模块** | **操作权限** | **提示词所有者** | **工作空间所有者、管理员** | **工作空间普通成员** |
| --- | --- | --- | --- | --- |
| 编辑提示词 | 编辑本人创建的提示词 | ✅ | ✅ | ✅ |
|  | 编辑工作空间内其他成员创建的提示词 | ❌ | ✅ | ❌ |
| 查看、复制、引用提示词 | 查看、复制、引用本人创建的提示词 | ✅ | ✅ | ✅ |
|  | 查看、复制、引用工作空间内其他成员创建的提示词 | ✅ | ✅ | ✅ |
| 删除提示词 | 删除本人创建的提示词 | ✅ | ✅ | ✅ |
|  | 删除工作空间内其他成员创建的提示词 | ❌ | ✅ | ❌ |
